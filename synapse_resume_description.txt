SYNAPSE — Network Port Scanner (Rust)
======================================

A high-performance TCP port scanner written in Rust, inspired by Nmap. Performs
network reconnaissance via stealth SYN scanning and TCP connect scanning, with
built-in host discovery, reverse DNS resolution, and service identification.

FEATURES
--------
- Dual-mode scanning: raw SYN (stealth) and full TCP connect scans
- ICMP host discovery with "blast-and-collect" timing strategy
- Reverse DNS (PTR) lookups via a hand-rolled binary DNS implementation (RFC 1035)
- Service name mapping using the nmap-services database
- Flexible target specification: single IPs, CIDR blocks, hostnames, and octet
  ranges (e.g., 10.0.0.1-50)
- Full IPv4 and IPv6 dual-stack support
- Optional timestamped file output of scan results
- Safety guardrails: hardcoded exclusion of major public DNS infrastructure

TECHNOLOGIES & CRATES
----------------------
- Rust (edition 2024)
- tokio          — async runtime for concurrent TCP connect scanning
- pnet           — raw socket access and transport-layer packet construction
- clap           — CLI argument parsing with derive macros
- ipnetwork      — CIDR block parsing and IP enumeration
- rand           — cryptographically seeded RNG for TCP sequence numbers
- indicatif      — terminal progress bars during host discovery

TECHNIQUES
----------
- Raw Socket Programming: manually constructs TCP and ICMP packets at Layer 4
  using pnet transport channels; handles IPv4 and IPv6 checksum calculation
- SYN Scanning: sends raw SYN packets, captures replies, classifies port state
  from TCP flags (SYN-ACK = Open, RST = Closed, timeout = Filtered), and sends
  RST to quietly tear down half-open connections
- Async Concurrency: spawns per-port tokio tasks gated by a Semaphore for
  controlled parallelism (configurable, default 1000 concurrent tasks)
- Binary DNS Protocol: builds RFC 1035 DNS queries from scratch over UDP and
  manually parses binary responses including label-compression pointer resolution
- ICMP Discovery: rapid-fires echo requests to all targets, then collects replies
  within a 3-second deadline using a HashMap to track host liveness
- Target Parsing: multi-format input parser handles CIDR, hostname (async DNS
  resolution), and octet-range notation with warning thresholds for large scans
- Linux Capabilities: uses CAP_NET_RAW (setcap) to enable raw socket operations
  without requiring a full root process

ARCHITECTURE
------------
src/main.rs           — CLI orchestration, output formatting, emit! macro
src/scanner.rs        — SYN scan and connect scan implementations
src/packet.rs         — raw packet construction, transport channel management
src/host_discovery.rs — ICMP discovery with progress tracking
src/utils.rs          — target/port parsing, DNS resolver, service loader, I/O
data/nmap-services    — port-to-service name database
